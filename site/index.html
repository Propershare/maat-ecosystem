The Seven Pillars

The structural foundation of the ecosystem.
01

MAAT Core

Schemas + kernel. The constitutional layer. 7 JSON schemas define identity, memory, task, policy, event, tool, and learning. The kernel has exactly 8 jobs. Nothing more.

02

MAAT Memory

Five real memory classes: episodic, semantic, constitutional, task, working. Constitutional memory is append-only, versioned, and never silently overwritten. Consolidation promotes episodic to semantic.

03

MAAT Policy

Machine-readable moral and operational constitution. Five outcomes: allow, deny, escalate, require_approval, log. Three-Ring governance. Fail-closed. Prompts are sand. Policies are stone.

04

MAAT Tools

Every action through a standard contract. Tools declare their name, parameters, required ring, and reversibility. Ring-based access control enforced by the policy engine.

05

MAAT Learning

Controlled adaptation. Always reversible. Snapshots are mandatory. If learning corrupts the system, rollback restores prior state. Rolling back is the system working correctly.

06

MAAT Runtime

Kernel + adapters + event bus. Single or multi-agent. Events are the nervous system — not logs, not print statements. Every meaningful action emits a structured event.

07

MAAT Commons

Apps, packs, policies, skills — the downloadable ecosystem. Install a policy pack, swap a tool pack, download a role. Without ripping open the engine.

Sacred vs Replaceable

If you cannot tell which side something falls on, it is replaceable.

◆ Sacred

  • Identity schema
  • Memory class definitions (5 classes)
  • Policy semantics (5 outcomes)
  • Event taxonomy (namespaced)
  • Task lifecycle states (6 states)
  • Tool contract shape
  • Three-Ring governance
  • Constitutional memory: append-only
  • Learning: always reversible
  • Events: append-only log

○ Replaceable

  • Model provider (Ollama, OpenAI, Anthropic, vLLM, Groq)
  • Vector DB (pgvector, Qdrant, Chroma, Pinecone)
  • SQL backend (PostgreSQL, SQLite, DuckDB)
  • Tool transport (MCP, REST, gRPC, native)
  • Embedding model (nomic, OpenAI, Cohere)
  • CLI framework (argparse, Click, Typer)
  • UI framework (Flask, React, htmx, terminal)
  • Agent runtime (OpenClaw, LangChain, CrewAI, AutoGen)
  • Messaging transport (Telegram, Discord, WhatsApp)

Doctrine

Five documents that define the rules of the ecosystem.

CONSTITUTION.md

What is sacred and what is replaceable. The amendment process. If you cannot tell which side something falls on, it is replaceable.

PORTABILITY.md

The migration guarantee. Identity survives runtime changes. Memory survives backend changes. Policies survive UI changes. The 9-question compliance test.

EVENTS.md

Canonical event taxonomy. 40+ typed events across 10 domains. Events are the nervous system — not logs, not print statements. Append-only.

POLICY.md

5 outcomes. Evaluation order. Three-Ring governance. Fail-closed. Escalation paths. Policy vs prompts: prompts are sand, policies are stone.

LEARNING.md

Learning must be reversible. Snapshots mandatory. The observe→propose→approve→apply→verify→rollback loop. Rolling back is not failure.

Philosophy

Built on stable meaning, not new tech.
The contracts are at the center, not the tech.
Prompts are sand. Policies are stone.
Constitutional memory is never silently overwritten. It is amended through traceable versioning.
If learning is not reversible, the system becomes self-corrupting.
Identity must outlive implementation.
If it happened and no event was emitted, it didn't happen.
Apps should be manifests, workflows, and policies — not secret business logic that bypasses the core.
Rolling back is not failure. Rolling back is the system working correctly.
Truth · Balance · Order · Justice · Self-Reflection

## Architecture (mental model)

- **Layers:** OpenClaw = client/runtime (channels, gateway, model); ecosystem = Discovery + organ MCPs + Postgres + soul files.
- **Learning:** Persists when tools hit **gitMaat / memory** (`log_*`, `query_*`), not “model memory” alone.
- **Heartbeat:** (A) **`/health`** = organ liveness; (B) **agent HEARTBEAT.md / gitMaat task poll** = coordination attention — complementary, not one daemon.

## MaatBench

- Lives in **`maat-ecosystem/maatbench/`** — **contract/system doctor**, not an HTTP organ on Discovery. **`maatbench.run`** mostly non-LLM tiers; **`scripts/maatbench-llm-evaluation.py`** = **Ollama** model names for LLM-side eval. Live “model + ecosystem tools” harness = separate integration.

## Repo / clone / who needs it

- **Git clone does not auto-discover MCPs** — need **known host/IP** + client MCP config + **Bearer** if enabled.
- **Primary holder of repo:** operator / machine that **runs and maintains** the body (system of record). **Agents** usually need **endpoints + trust**, not the monorepo. Optional: second device, reinstall, backup — still **same operator**.

## Tehuti Guard vs bootstrap UX

- **Guard** = governance (**approve/block/propose**, three-ring) — **not** key generation or one-click client wiring.
- **Stated product gap:** **local** API **key issue + rotation**, minimal / **non-dev** setup (“put it in place” on **own LAN**), appliance-style — **friction = debt** to remove over time.

## North star

- **Maat Ecosystem** should **behave as Maat**: honest about state, **single spine**, memory that **compounds**, health visible, **Guard + bench + constitution** as **evolution feedback** — not a static folder tree.

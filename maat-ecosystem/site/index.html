Sacred vs Replaceable

If you cannot tell which side something falls on, it is replaceable.

◆ Sacred

  • Nine organs — the body plan
  • MANIFEST.ka — the DNA
  • Boot sequence — soul first
  • Constitutional memory: append-only
  • Policy semantics (5 outcomes)
  • Identity schema
  • Memory class definitions (4 types)
  • Event taxonomy (namespaced)
  • Learning: always reversible
  • Three-Ring governance

○ Replaceable

  • Model provider (Ollama, OpenAI, Anthropic)
  • Memory backend (Postgres, SQLite, Redis, Pinecone)
  • Tool transport (MCP, REST, gRPC, native)
  • Voice channel (Terminal, Web, Telegram, Discord)
  • Blood bus (n8n, Redis Streams, NATS, RabbitMQ)
  • Ka monitor (systemd, Docker, K8s probes)
  • Senses input (webhooks, MQTT, file watch, cron)
  • Skeleton format (JSON Schema, Protobuf, Avro)
  • Agent runtime (OpenClaw, LangChain, CrewAI)

Philosophy

Built on Kemetic philosophy, not Silicon Valley trends.
Build bodies, not machines. Organs, not microservices.
The soul boots first. An agent cannot act before it knows its moral constraints.
Prompts are sand. Policies are stone. Constitutions are bedrock.
If it hurts and nothing noticed, the body has no immune system.
Constitutional memory is never silently overwritten. It is amended through traceable versioning.
Identity must outlive implementation. The soul survives every upgrade.
If learning is not reversible, the system becomes self-corrupting.
The same body plan is meant to scale from a chatbot to a robot to a building — vocabulary is living-system; contracts are engineering.
Rolling back is not failure. Rolling back is the immune system working correctly.

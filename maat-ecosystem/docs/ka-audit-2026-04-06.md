## What Needs Attention

### 🔴 Critical — The Ka Organ (Immune System)

The Ka organ (`ka/`) has `pulse.yaml` config but NO running process. There is nothing that:
- Periodically checks all organ health
- Detects when an MCP goes down
- Auto-restarts failed services
- Tracks error patterns over time
- Predicts failures from history

**Recommendation:** Build `ka-monitor` — a lightweight daemon that:
- Pings each MCP endpoint on the pulse interval
- Logs pain events to maat-memory via the new MCP
- Attempts healing (systemctl restart) on failure
- Escalates to notification (Telegram) after threshold
- Runs as a systemd service itself

### 🟡 Important — Self-Learning Loop

Memory stores learnings but nothing acts on them automatically.

**Recommendation:** Build a **Learning Loop** process:

```
1. COLLECT  — Agent logs learnings during work (already works)
2. PATTERN  — Consolidator finds clusters (exists in code, not running)
3. PROMOTE  — High-confidence learnings → update policies/constitution
4. APPLY    — Brain consults learnings before decisions
5. MEASURE  — Track if applied learnings improve outcomes
6. DECAY    — Low-confidence learnings that never help → archive/forget
```

This is the body getting smarter over time. The `memory/patterns/consolidator.py` already exists — it needs to run on a schedule and feed results back to the brain.

### 🟢 Future — RAD System Endpoints

**RAD = Read, Act, Decide** — standardized endpoints every organ exposes:

```
/{organ}/read    → GET current state (health, config, stats)
/{organ}/act     → POST trigger an action
/{organ}/decide  → POST ask organ to make a judgment
/{organ}/health  → GET Ka-compatible health check
/{organ}/pain    → GET current pain/error state
```

Every MCP server would implement these 5 endpoints. The Ka organ queries them uniformly. Any new organ plugs in by implementing the RAD interface.

Example:
```
/memory/read     → current stats, recent activity
/memory/act      → store, search, consolidate
/memory/decide   → "should I remember this?" (confidence threshold)
/memory/health   → connection status, latency, capacity
/memory/pain     → recent errors, failed queries

/brain/read      → loaded models, active reasoning chains
/brain/act       → think, generate, classify
/brain/decide    → "which model for this task?"
/brain/health    → model availability, response times
/brain/pain      → hallucination rate, timeout history
```
